# ALNS 中固定 Gamma=3 的两策略对比实验方案

## 1. 目标与范围
本方案仅比较两种策略，且统一使用 Bertsimas-Sim 预算不确定集参数 `\Gamma=3`：
1. 前置鲁棒嵌入（Pre-embedded Robust ALNS）。
2. Robust Verification（后验鲁棒复核）。

本方案不再做 `\Gamma` 消融，也不再设置名义/组合组，目标是直接回答：在同等鲁棒预算强度下，两种策略谁更稳、谁更快。

## 2. 统一鲁棒不确定性设定（Gamma 固定为 3）

### 2.1 能耗不确定性
对任意弧 `a`：

\[
e_a = \bar e_a + \hat e_a \xi_a,
\quad 0 \le \xi_a \le 1
\]

### 2.2 Bertsimas-Sim 预算不确定集
对路线段 `\mathcal A_{r,s}`（如两次充电间）：

\[
\sum_{a \in \mathcal A_{r,s}} \xi_a \le \Gamma,
\quad \Gamma = 3
\]

### 2.3 鲁棒能量对应式

\[
\sum_{a\in\mathcal A_{r,s}} \bar e_a + \beta_{r,s}(3) \le B
\]

\[
\beta(3)=\hat e_{(1)}+\hat e_{(2)}+\hat e_{(3)}
\]

其中 `\hat e_{(k)}` 为该段偏差降序第 `k` 大值。

## 3. 两个对比方法的统一定义

### 3.1 方法 A：前置鲁棒嵌入（Pre-embedded Robust ALNS）
鲁棒性直接进入 ALNS 搜索过程：
1. 可行性检查按鲁棒能量约束执行（`\Gamma=3`）。
2. 邻域增量代价采用鲁棒目标 `\Delta F`。
3. 接受准则与算子打分均基于鲁棒目标。

建议目标函数：

\[
F(x)=c_{travel}(x)+\lambda\,viol_{time/cap}(x)+\mu\,viol_{energy\_rob}^{(\Gamma=3)}(x)
\]

### 3.2 方法 B：Robust Verification（Post-check ALNS）
搜索主体使用常规（或弱鲁棒）评价，候选解生成后再做 `\Gamma=3` 鲁棒复核：
1. 每次接受前或每 `k` 次迭代执行一次鲁棒复核。
2. 复核失败则 `hard reject` 或增加惩罚（需预先固定一种机制）。
3. 鲁棒复核计算与方法 A 使用同一 `\Gamma=3` 定义，确保公平对比。

## 4. 公平性控制（必须固定）
1. 同一实例集、同一随机种子集合。
2. 同一 destroy/repair 算子池。
3. 同一迭代预算、时间上限、停止条件。
4. 同一参数：`\lambda, \mu, T_0, cooling`。
5. 唯一变量：鲁棒性注入位置（前置嵌入 vs 后验复核）。

## 5. 实验指标（仅围绕两策略）

### 5.1 效率指标
1. 总运行时间 `T_total`。
2. 每 1000 次迭代耗时 `T_1000`。
3. 邻域评估平均耗时 `T_eval`。

### 5.2 质量与鲁棒性指标
1. 最终总成本（含惩罚）`F_final`。
2. 鲁棒可行率（`\Gamma=3` 检验通过比例）。
3. 最坏场景超能耗量（越小越好）。
4. 可选：CVaR 风险指标。

### 5.3 搜索行为指标
1. 接受率。
2. 有效改进频率。
3. 停滞长度。
4. 对方法 B：复核拒绝比例。

## 6. “是否加速”的判据

以方法 A 为基线，方法 B 为待检验策略：

\[
Speedup_{B/A}=\frac{T_A}{T_B}
\]

并报告单位时间改进效率：

\[
EffGain = \frac{F_{init}-F_{final}}{T_{total}}
\]

判定建议：
1. `Speedup_{B/A} > 1`：B 在时间上更快。
2. 同时 `F_final` 与鲁棒可行率不劣于 A：才可称“有效加速”。
3. 若更快但鲁棒性显著下降，只能称“以稳健性换速度”。

## 7. 统计与报告模板
1. 每方法至少 `N>=20` 随机种子重复。
2. 配对统计：Wilcoxon signed-rank（优先）或配对 t-test。
3. 报告：均值、标准差、95% CI、效应量。
4. 图表建议：
   - 箱线图：`T_total`、`F_final`。
   - 散点图：时间 vs 鲁棒可行率。
   - 轨迹图：迭代过程 `F(x)` 收敛曲线。

## 8. 落地执行清单（两组版）
1. 固定 `\Gamma=3` 与全部共享参数。
2. 实现方法 A（前置鲁棒嵌入）并作为主基线。
3. 实现方法 B（后验 robust verification），固定复核触发频率与拒绝规则。
4. 跑同种子批次并导出配对结果表。
5. 按“速度-鲁棒可行率-总成本”三维给出最终结论。
